## Title Tag Standards

### Formula
- **Core pages:** `[Primary Keyword] — Tacklers Consulting Group`
- **Service pages:** `[Service Name] Consulting UK — Tacklers`
- **Industry pages:** `[Industry] Operational Excellence — Tacklers`
- **Blog posts:** `[Post Title — max 48 chars] — Tacklers Blog`
- **Conversion pages:** `[Action — Descriptor] — Tacklers`
- **Legal pages:** `[Page Name] — Tacklers Consulting Group`

### Rules
1. Maximum 60 characters (including brand suffix)
2. Primary keyword within first 4 words where possible
3. Brand name always last
4. Use em dash (—) as separator
5. British English throughout

### Rules
1. Maximum 155 characters
2. Includes primary keyword naturally
3. Ends with a CTA or action prompt
4. British English throughout
5. No duplicate descriptions across pages

## Open Graph Standards

### Rules
1. `og:title` — mirrors page title
2. `og:description` — mirrors meta description
3. `og:image` — page-specific image (1200×630px) for top 10 pages; generic brand image for others
4. `og:image:alt` — descriptive alt text for every og:image
5. `og:type` — "website" for pages, "article" for blog posts
6. `og:url` — canonical URL
7. `twitter:card` — "summary_large_image"
8. `twitter:site` — company Twitter handle

### Priority pages for custom og:image
1. Home
2. Services pillar
3. Mentoring
4. About
5. Blog hub
6. Each new pillar page (Lean Transformation, CI, Gemba)
7. Case studies hub

## Structured Data Standards

### Required per page type
| Page type | Required schemas |
|-----------|-----------------|
| All pages | Organization (via layout), WebPage, BreadcrumbList |
| Home | + LocalBusiness, FAQPage, WebSite (without SearchAction) |
| Services pillar | + Service (per card), FAQPage |
| Service detail | + Service, FAQPage |
| Industry | + FAQPage |
| Blog post | + BlogPosting (with author, datePublished, dateModified) |
| Blog hub | + CollectionPage |
| Case study | + Article or BlogPosting |
| Contact | + LocalBusiness (with address) |
